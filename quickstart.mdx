This guide will help you deploy DeepLX and make your first translation request.

## Run DeepLX

<Steps>
  <Step title="Start with Docker">
    The fastest way to run DeepLX is using Docker:

    ```bash
    docker run -d -p 1188:1188 missuo/deeplx
    ```

    This starts DeepLX on port 1188 (default) listening on all interfaces.

    <Accordion title="Configuration Options">
      DeepLX supports several environment variables and flags (service/config.go:21-75):

      | Option | Environment Variable | Flag | Default | Description |
      |--------|---------------------|------|---------|-------------|
      | IP Address | `IP` | `-ip`, `-i` | `0.0.0.0` | Bind address |
      | Port | `PORT` | `-port`, `-p` | `1188` | Listen port |
      | Access Token | `TOKEN` | `-token` | `""` | Authentication token |
      | DL Session | `DL_SESSION` | `-s` | `""` | Pro account session |
      | Proxy | `PROXY` | `-proxy` | `""` | HTTP proxy URL |

      **Example with authentication:**
      ```bash
      docker run -d -p 1188:1188 -e TOKEN=your_secret_token missuo/deeplx
      ```

      **Example with custom port:**
      ```bash
      docker run -d -p 9000:9000 -e PORT=9000 missuo/deeplx
      ```
    </Accordion>
  </Step>

  <Step title="Verify the service is running">
    Check that DeepLX is responding:

    ```bash
    curl http://localhost:1188/
    ```

    Expected response (service/service.go:101-106):
    ```json
    {
      "code": 200,
      "message": "DeepL Free API, Developed by sjlleo and missuo. Go to /translate with POST. http://github.com/OwO-Network/DeepLX"
    }
    ```
  </Step>

  <Step title="Make your first translation request">
    Translate text from English to German:

    ```bash
    curl -X POST http://localhost:1188/translate \
      -H "Content-Type: application/json" \
      -d '{
        "text": "Hello, World!",
        "source_lang": "EN",
        "target_lang": "DE"
      }'
    ```

    <CodeGroup>
      ```bash cURL
      curl -X POST http://localhost:1188/translate \
        -H "Content-Type: application/json" \
        -d '{
          "text": "Hello, World!",
          "source_lang": "EN",
          "target_lang": "DE"
        }'
      ```

      ```python Python
      import requests

      response = requests.post(
          "http://localhost:1188/translate",
          json={
              "text": "Hello, World!",
              "source_lang": "EN",
              "target_lang": "DE"
          }
      )

      print(response.json())
      ```

      ```javascript JavaScript
      const response = await fetch("http://localhost:1188/translate", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          text: "Hello, World!",
          source_lang: "EN",
          target_lang: "DE"
        })
      });

      const data = await response.json();
      console.log(data);
      ```
    </CodeGroup>
  </Step>

  <Step title="Review the response">
    A successful translation returns (service/service.go:134-142):

    ```json
    {
      "code": 200,
      "id": 8912375012,
      "data": "Hallo, Welt!",
      "alternatives": [
        "Hallo Welt!",
        "Servus Welt!"
      ],
      "source_lang": "EN",
      "target_lang": "DE",
      "method": "Free"
    }
    ```

    **Response fields:**
    - `code`: HTTP status code
    - `id`: Unique request identifier
    - `data`: Primary translated text
    - `alternatives`: Alternative translation options
    - `source_lang`: Detected or specified source language
    - `target_lang`: Target language code
    - `method`: Translation method used (`Free` or `Pro`)
  </Step>
</Steps>

## Request Parameters

The `/translate` endpoint accepts (service/service.go:63-68):

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `text` | string | Yes | Text to translate |
| `source_lang` | string | No | Source language code (auto-detected if omitted) |
| `target_lang` | string | Yes | Target language code |
| `tag_handling` | string | No | Handle HTML/XML tags: `"html"` or `"xml"` |

<Note>
  When `source_lang` is empty or `"auto"`, DeepLX automatically detects the source language (translate/translate.go:120-122).
</Note>

## Authentication (Optional)

If you configured DeepLX with a `TOKEN`, include it in requests:

**Query parameter:**
```bash
curl -X POST "http://localhost:1188/translate?token=your_secret_token" \
  -H "Content-Type: application/json" \
  -d '{"text": "Hello", "target_lang": "DE"}'
```

**Authorization header (service/service.go:33-46):**
```bash
curl -X POST http://localhost:1188/translate \
  -H "Authorization: Bearer your_secret_token" \
  -H "Content-Type: application/json" \
  -d '{"text": "Hello", "target_lang": "DE"}'
```

<Info>
  DeepLX supports both `Bearer` and `DeepL-Auth-Key` authorization formats for compatibility.
</Info>

## Error Handling

<Warning>
  If you see HTTP 429, your IP has been temporarily blocked by DeepL for making too many requests (translate/translate.go:78-80). Wait before retrying.
</Warning>

Common error responses:

**Invalid token:**
```json
{
  "code": 401,
  "message": "Invalid access token"
}
```

**Invalid tag_handling:**
```json
{
  "code": 400,
  "message": "Invalid tag_handling value. Allowed values are 'html' and 'xml'."
}
```

**No text provided:**
```json
{
  "code": 404,
  "message": "No text to translate"
}
```
